## 4-2 使用 Browser 測試網頁應用程式

### 下載「旗標購物網」專案

- [旗標購物網的 GitHub 網站](https://github.com/FlagTech/Shopping_Demo)

- 下載命令：

```md
git clone https://github.com/FlagTech/Shopping_Demo

cd Shopping_Demo

uv run app.py
```

### 進行自動化測試

- 輸入的提示內容：

```md
請執行標準的網站測試流程：
1. 使用 @Browser  導航到 http://127.0.0.1:5000
2. 填寫註冊表單並登入：
   -電子郵件：test@example.com - 密碼：testpass1233 
3. 驗證首頁頁面載入
4. 測試按鈕功能及各種響應式設計
5. 檢查控制台錯誤
6. 截取螢幕截圖進行視覺回饋測試
請先記錄所發現的錯誤，撰寫成 md 檔
```

### 設定自動化測試的 Agent

- 輸入的提示內容：

```md
請建立一個使用 @Browser 進行自動化測試的 Agent，撰寫測試規範並儲存到 .cursor\rules 中的 testing.mdc。先說明清楚測試環境設定、測試資料和偵測標準，然後列出主要測試項目，每個項目需要有明確的目的、測試步驟及檢查項
```

- 輸入的提示內容：

```md
@.cursor/rules/testing.mdc 請執行測試項目 4：商品篩選功能測試
```

#### 建立新的測試項目

- 輸入的提示內容：

```md
@.cursor/rules/testing.mdc 幫我新增「結帳表單驗證」的測試項目
```


### 除錯處理

- 輸入的提示內容：

```md
[@測試報告] 根據測試報告的內容，進行除錯。
```

- 輸入的提示內容：

```md
@.cursor/rules/testing.mdc 根據除錯的內容，重新進行測試
```


## 4-3 Figma MCP－設計前端介面的好幫手

### 參考設計風格

- [Figma 網站](https://www.figma.com/)

- 輸入的提示內容：

```md
[Figma link] 請使用 Figma MCP，參考此設計風格來修改首頁的 Nav Bar 和 Page Header，並保留我原先的文案
```

### 選取元素來進行設計

- 輸入的提示內容：

```md
使用 Figma MCP，參考 [Figma link] 的配色風格來調整 @元件 的版面設計，並保留原本文案
```

### 利用 Agent 來實作能重複使用的元件

- 輸入的提示內容：

```md
請建立一個實作 UI 元件的 Agent，此 Agent 會使用 Figma MCP 來參考 UI 元件的設計，然後將設計轉換成可重複使用的 UI 元件並儲存到 ‘templates/components/’ 中。詳細描述此 Agent 的規則並儲存到 ‘.cursor/rules/ui_design.mdc‘ 中
```

- 輸入的提示內容：

```md
使用 @.cursor/rules/ui_design.mdc 保存 [Figma link] 的按鈕設計
```

- 輸入的提示內容：

```md
將 [@UI 元件] 替換為 [@<模板>.html] 中的綠色按鈕設計
```
